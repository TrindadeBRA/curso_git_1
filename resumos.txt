O que é Controle de versão?
- Gerencia o código-fonte;
- Registra todas alterações de código;
- Criar versões de um software estágios; 
- Cada membro pode trabalhar em uma versão diferente (branch);
- Ferramentas: git e SVN.

O que é Git?
- Sistema de versionamento mais utilizando no mundo;
- Baseado em repositórios, contem todas as versoes do codigo e as cópias dos devs;
- Alto desempenho;
- Protegidos como criptografia;

O que é um repositório?
- É onde o código vai ser armazenado;
- Cada projeto em um repo;
- Gerenciar repos em: GitHub / BitBucket;
- Cada dev pode baixar o repo e criar sua versão;

O que é um GitHub?
- Serviço para gerenciar repos gratuitamente;

OBS: "Q ou CTRL/C" devolve o terminal! 
OBS: "clear" limpa o terminal! 

Como inicializar o git?
- Código: " git init ";

Como verificar mudanças do projeto?
- Código: " git status ";

Adicionando arquivos no projeto!
- Código: " git add ";
- Código: " git add . " (Adiciona todos os arquivos no projeto);

Salvando alterações do projeto!
- Código: " git commit ";
- Código: " git commit -m "Mensagem" " // Para poder commitar com mensagem ;
- Código: " git commit -a " // Para poder commitar todos os arquivos ;
- Código: " git commit arquivo.txt -m "Enviando um arquivo de texto" " // Para poder commitar apenas um arquivo com mensagem ;

Enviando as alterações ao repo remoto!
- Código: " git push "; //Empurrar alterações

Recebendo as mudanças!
- Código: " git pull "; //puxar alterações

Clonar repositório!
- Código: " git clone https://github.com/TrindadeBRA/curso_git_1.git ." // O Ponto no final, é para clonar no diretorio que está;

Removendo / Deletar arquivos do repo!
- Código: " git rm nomedoarquivo.txt ";

Histórico de alterações!
- Código: " git log ";

Mover ou Renomear arquivos!
- Código: " git mv ";

Desfazendo alterações!
- Código: " git checkout nomedoarquivo.txt ";

Ignorando arquivos!
- .gitignore

Desfazendo as alterações!
- Código: " git reset ";


Branches!
- Forma que o git separa as versões dos projetos;
- Cada nova feature de um projeto fica em um branch separado;
- Depois de finalizar as feature, as branchs devem ser unidas (merge);

Criando Branches!
- Código: " git branch "; // Visualizar branchs
- Código: " git branch <nome> ";

Deletar Branches!
- Código: " -d ou --delete <nome>";
- Normalmente não é deletado um branch;

Mudando de branch!
- Código: " git checkout -b <nome>";

Unindo branches!
- Código: "git merge <nome>"

Usando Stash!
- Joga o codigo novo da branch no "lixo";
- Código: " git stash ";

Recuperando stash!
- Código: " git stash list "; // lista stashs
- Código: " git stash list show "; // lista stashs e lista alteracoes
- Código: " git stash list <numero> "; // resgata

Removendo as stash!
- Código: " git stash clear "; // limpa todas stachs
- Código: " git stash drop <numero> "; // limpa stachs especifica

Utilizando tags!
- Criar tags nas branchs; 
- Serve como checkpoint;
- Código: " git tag -a <nome> -m <msg> ";
